MLOps Pipeline - Pretrained Vision + Fleet Knowledge Sync

New target architecture for an image-only pretrained detector running on a live camera feed. The model stays frozen, embeddings remain preloadable, unknown objects are labeled by the operator in the UI, and validated knowledge is synchronized across devices through a fleet-level MLOps control plane.

Frozen pretrained model

The production robot does not retrain the backbone. Day-to-day learning happens by updating embeddings, labels, thresholds, and knowledge-pack metadata around the frozen model.

Buffered video to image inference

The camera feed is buffered and sampled into still frames because the deployed model processes images, not native video streams.

Operator in the loop

Unknown detections are routed into a manual labeling queue in the UI so new objects become curated data instead of silent model errors.

Embedding preload stays

Boot-time preload of saved embeddings remains part of the architecture, giving each device immediate known-object capability before any new session data is collected.

Fleet-wide knowledge sync

Validated labels and embeddings are promoted as versioned knowledge releases and synchronized across devices through the fleet sync hub.

MLOps-complete without forced retraining

The required MLOps building blocks are still present: data lake, versioning, orchestration, feature stores, experiment tracking, registries, and monitoring, while full model training is pushed upstream.
